Ruby classes vs. JavaScript constructor functions

The majority of object-oriented languages use classes to create objects and define the behavior of that object. Ruby is an object-oriented language that does, while JavaScript is an object-oriented language that does not. Classes may be objects themselves, but if so they're different from other types of objects. Inside a class object you will find all the methods needed to declare an instance of that class i.e an object, as well as all the functional behaviors that each object inherits by being declared in that class, in other words you'll find instance variables and instance methods. The initialize method is the method that actually creates new methods. It is in this function where you can have default values for instance variables whereby each object will get a separate copy of that variable.

Ruby Class Class Employee def initialize(name, salary, hire_year) @name = name @salary = salary @hire_year = hire_year end def to_s "Name is #{@name}, salary is #{@salary}" "hire year is #{hire_year}" end def raise_salary_by(perc) @salary += (@salary * 0.10) end end

JavaScript has objects but no classes. Thus there's no special syntax for defining how to create an instance or define its behavior. JavaScript objects (or instances) are created with a constructor. The constructor of an object (or instance) is a function that is inovoked with the new operator. In JavaScript, any function can be a constructor--even though it may not look like one.

How does JavaScript define behavior of objects? JavaScript doesn't have a special syntax or special kind of object for that, it has instead uses functions to double as object constructors also called prototypes. Constructors have no special methods or properties associated with them. Any object can be a constructor (prototype), even a DOM object and an empty object. An empty object is actually what is associated with a constructor by default.
